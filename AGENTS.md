# Project instructions

This is a TypeScript NestJS portfolio project.

Use:
- NestJS modules/services/controllers pattern
- Prisma for database access
- PostgreSQL
- DTOs with class-validator
- Swagger decorators
- Jest for tests

Rules:
- Do not change public API contracts without asking.
- Always add tests for new services.
- Keep business logic in services, not controllers.
- Use strict TypeScript.
- Run npm test and npm run lint after changes.

# Test rule

Whenever you create a new NestJS service, add a comment:

// AGENT_TEST_2026

at the top of the file.
